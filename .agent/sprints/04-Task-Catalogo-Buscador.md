# Sprint 4: Buscador Estratégico y Visualización de Catálogo
**Objetivo:** Construir la experiencia de producto y volumen de precio (B2B).

**Dependencias previas:** Interfaces creadas en 02-Task, Layout creado en 03-Task.

**Acciones:**
1. **Buscador Mixto:** Crea un motor/input de búsqueda avanzado que filtre tanto por *Nombre de la Pieza* como por el campo *SKU* en la base de datos de los productos en pantalla.
2. **Exhibición (Volume Pricing):** Las 'cards' de los productos deben incluir obligatoriamente etiquetas visuales ("Llévate x caja entera a descuento" / "Consulta mayorista") tal cual manda el PDF.
3. Componente individual: un modal o ruta `product/[id]` limpia mostrando fotos (LED, herramientas) en alta resolución con descripción corta.

**Criterios de Éxito:** Interacción fluida para buscar (ej. tipear el sku 'TF-220V' y que el grid filtre en tiempo real). UI atractiva para motivar las compras voluminosas.
